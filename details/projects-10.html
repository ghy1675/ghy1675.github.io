상명대학교 투표 시스템 취약점 제보

Project information

  • Category: Python3, Scapy
  • Project date: Jun. 2021 ~ Jun. 2021
  • Result: 학교 제보

Project Details

2021년도 1학기 캡스톤 전시회를 온라인으로 진행했습니다. 온라인 페이지에는 각 단과대학별로 인상깊거나 잘했다고 생각하는 팀에게 하트를 눌러 투표하고 최다 득표자에게 시상하는 시스템이 있었습니다.

공대부문의 타 팀의 프로젝트를 확인하다가 조회수 보다 투표수가 월등히 높은 팀들이있었습니다. 그래서 이 팀들은 뭔가 편법을 썼다 라는 생각으로 연구를 시작했습니다.
이 투표시스템을 사용하던 중 크롬 관리자 도구를 이용해 패킷에 어떤 값이 담겨있나 확인해보다가 연구를 진행했습니다.

본 프로젝트에서 팀장을 맡아서 현재 발견한 내용을 팀원들과 공유하여 실험 시나리오 및 PoC를 작성하여 공유하고 나온 결과를 보고서로 작성하여 학교에 제출하였습니다.

본 프로젝트는 Python3로 진행했고 주로 사용한 패키지는 다음과 같습니다.
- requests : REST API 요청

이전 프로젝트와 비슷하게 Breakthrough는 다음과 같은 순서로 진행 되었습니다.
1. 대상 URL 구조 파악
2. 쿠키, 세션 사용 여부 및 구조 파악
3. 투표 버튼 클릭시 전달 되는 패킷 구조 파악
4. PoC 작성

초기에는 인증 여부를 확인하며 패킷을 하나씩 생성하여 전송했고, 투표자에 대한 인증과 인가가 없었다는 점을 파악하였습니다.
다음으로는 개별 패킷에서 대량의 패킷을 전송하도록 코드를 고도화 하였고, 자동화 공격에 대한 임계점을 파악하였고, 임계점에 도달 할 때 마다 IP를 변경하며 실험을 진행했습니다.

발견한 내용들을 다른 사람들이 더 악용하기전에 학교에 제보하여 막아야겠다는 생각으로 학교에 제보하였습니다.
이후 학교 측에서는 해당 사실을 공지하였고, 비정상적으로 투표수가 높은 팀이 많아 공대는 시상에서 아예 제외하였습니다.
