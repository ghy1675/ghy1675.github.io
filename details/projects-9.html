대학교 E-Campus내의 취약점 제보

Project information

  • Category: Python3, Scapy
  • Project date: Feb. 2021 ~ Feb. 2021
  • Result: KISA 제보

Project Details

일반적으로 E-Campus 내부에 업로드 된 강의 자료는 해당 강의를 수강하는 학생만 다운로드 받을 수 있습니다
그렇다면 이 인증과정은 어떻게 되어있는가? 라는 궁금증과 이를 어떻게 우회할 수 있는 지에 대해 고민하면서 시작했습니다.
그 당시 재학중이던 학교의 E-Campus 인증 루틴 파악 후 타 학교 E-Campus의 UI와 URL이 구조가 본교와 동일했던 기억이 나서, 타 학교로 영역을 확장했습니다.

본 프로젝트에서 맡은 역할은 BreakThrough였습니다. E-Campus 내부의 취약점을 발굴하여 다른 팀원에게 전달하였습니다.

본 프로젝트는 Python3로 진행했고 주로 사용한 패키지는 다음과 같습니다.
- Multiprocess : 다량의 요청 패킷 생성
- requests : REST API 요청

Breakthrough는 다음과 같은 순서로 진행 되었습니다.
1. 대상 URL 구조 파악
2. 쿠키, 세션 사용 여부 및 구조 파악
3. 파일 다운로드시 전달 되는 패킷 구조 파악
4. PoC 작성

전국 대학 중 무작위로 60곳을 선정하여 조사했습니다.
상명대학교를 포함한 14개의 대학에서는 신원 인증없이 해당 URL만 입력하면 다운로드 가능했습니다.

위 내용은 각각 대학에 전달하는 것보다 KISA에 전달하는게 제일 효과적일것 같아 KISA에 제보하였습니다.
